# Описание торговых стратегий

Бот реализует две независимые стратегии. Каждая стратегия работает в отдельном потоке на одном и том же наборе тикеров, использует свою логику входа/выхода и сохраняет сделки с пометкой `strategy_name`. Управление стратегиями (включить/выключить) доступно в разделе **Стратегии** административной панели.

---

## 1. Combo: OFI + Large Print

**Тег в базе данных:** `combo`  
**Таймфрейм:** тиковые данные (обновления стакана и сделки в реальном времени)  
**Конфиг:** `trading_bot/config/instruments.yaml`

### Идея

Стратегия основана на двух одновременных сигналах:

1. **OFI (Order Flow Imbalance)** — дисбаланс потока заявок в стакане. Показывает, кто агрессивнее: покупатели или продавцы.
2. **Large Print (Крупный принт)** — крупная сделка, объём которой значительно превышает типичный. Это маркер того, что крупный участник занял или разгружает позицию.

Сигнал входа формируется только когда оба условия выполнены одновременно и направлены в одну сторону: OFI указывает на покупателей и фиксируется крупный покупатель — открываем лонг. OFI указывает на продавцов и фиксируется крупный продавец — открываем шорт.

### Алгоритм OFI

За основу взята модель Cont-Kukanov-Stoikov. При каждом обновлении стакана считается, как изменились объёмы на лучших N уровнях с каждой стороны:

- Если лучший бид вырос по цене или объёму — это давление на покупку (+)
- Если лучший аск вырос по цене или объёму — это давление на продажу (−)

Сырой OFI нормализуется через `tanh(raw / ofi_scale)` — результат всегда в диапазоне (−1, +1). Затем применяется скользящее среднее по `ofi_smooth_window` последним значениям.

**Подбор `ofi_scale`:** чем ликвиднее инструмент, тем больше значение. Ориентир: при `raw_ofi = ofi_scale` → нормализованный OFI ≈ 0.76. Если OFI почти всегда около нуля — `ofi_scale` слишком велик; если почти всегда ±1 — слишком мал.

### Алгоритм Large Print

Каждая входящая сделка сравнивается с медианой объёмов последних `print_window` сделок:

```
объём сделки >= медиана × print_multiplier  →  крупный принт
```

Направление принта (покупка/продажа) определяется из стрима — если биржа не указывает агрессора, используется тик-тест (сравнение цены с предыдущим бидом/аском).

Принт считается актуальным `print_max_age_seconds` секунд после фиксации. Устаревший принт при входе игнорируется.

### Условие входа

```
|OFI| >= ofi_threshold                 (достаточный дисбаланс)
OFI и принт совпадают по направлению  (подтверждение)
Принт не старше print_max_age_seconds  (актуальность)
Подряд min_ofi_entry_confirmations     (фильтр шума)
Цена выше MA(trend_ma_window) для лонга, ниже — для шорта  (фильтр тренда, если включён)
Cooldown с предыдущего входа прошёл    (защита от переторговки)
```

### Условие выхода

Стратегия мониторит выход по трём механизмам (первый сработавший закрывает позицию):

| Механизм | Условие |
|---|---|
| OFI-разворот | OFI уходит против позиции на `min_ofi_confirmations` подтверждений подряд. Выход блокируется если удержание меньше `min_hold_seconds` или прибыль меньше `min_profit_ticks_for_ofi_exit` |
| Стоп-лосс | Цена ушла против позиции на `stop_ticks` × `tick_size` |
| Безубыток | После движения на `breakeven_ticks` стоп переносится на цену входа (только при `trailing_stop_ticks = 0`) |
| Трейлинг-стоп | Стоп следует за ценовым экстремумом на расстоянии `trailing_stop_ticks` (приоритет над безубытком) |
| Тейк-профит | Движение в пользу позиции достигло `take_profit_ticks` (0 = отключён) |
| Тайм-аут | Позиция удерживается более `max_hold_minutes` минут |

### Ключевые параметры

| Параметр | Назначение |
|---|---|
| `ofi_threshold` | Минимальный OFI для входа (0.6–0.8) |
| `ofi_scale` | Масштаб нормализации, подбирается под ликвидность инструмента |
| `ofi_smooth_window` | Окно сглаживания OFI (больше = медленнее реагирует) |
| `print_multiplier` | Насколько крупный принт больше медианы (авто-калибровка через `calibrate_multipliers.py`) |
| `print_window` | Окно для расчёта медианы объёмов |
| `print_max_age_seconds` | Свежесть принта (15с для ликвидных, 25–30с для менее ликвидных) |
| `min_ofi_entry_confirmations` | Сколько баров подряд OFI должен быть выше порога (3–5) |
| `trend_ma_window` | Окно MA для фильтра тренда (0 = отключён) |
| `ofi_exit_threshold` | Порог OFI для выхода (обычно меньше порога входа) |
| `min_ofi_confirmations` | Подтверждений разворота OFI подряд (3–5) |
| `cooldown_seconds` | Пауза между входами (защита от переторговки) |

## 2. Augmented RSI 5m (LuxAlgo)

**Тег в базе данных:** `rsi`  
**Таймфрейм:** 5-минутные свечи (строятся из тиков в реальном времени)  
**Конфиг:** `trading_bot/config/rsi_config.yaml`

### Идея

Стратегия использует модифицированный RSI — **Augmented RSI**, разработанный командой LuxAlgo. В отличие от классического RSI, который считает только отношение средних движений вверх/вниз, Augmented RSI учитывает диапазон свечи (расстояние от максимума до минимума), что делает его более чувствительным к реальному импульсу.

Логика входа проста: рынок зашёл в зону перепроданности (RSI < 20) и затем начал выходить из неё — это сигнал для покупки. Аналогично в зоне перекупленности (RSI > 80) при начале выхода из неё — сигнал для продажи.

Такой подход ловит разворотные движения после истощения тренда, что хорошо работает в условиях внутридневной волатильности российского рынка.

### Алгоритм

Каждые 5 минут строится новая свеча (OHLCV). При закрытии свечи вычисляется:

```
upper = максимум цен закрытия за последние length свечей
lower = минимум цен закрытия за последние length свечей
r = upper − lower                  (диапазон)
d = close − close[−1]              (изменение за бар)

diff = r   если upper только что обновил максимум
     = −r  если lower только что обновил минимум
     = d   иначе (обычное движение внутри диапазона)

num    = RMA(diff, length)          (сглаженное накопленное движение)
den    = RMA(|diff|, length)        (сглаженная абсолютная активность)
arsi   = (num / den) × 50 + 50     (диапазон 0–100)
signal = EMA(arsi, smooth)          (сигнальная линия)
```

RMA (Wilder's Moving Average) — экспоненциальная с alpha = 1/length. Более инертная чем EMA.

### Условие входа

```
LONG:  arsi был < os_value (20) и теперь >= os_value  →  рынок выходит из перепроданности
SHORT: arsi был > ob_value (80) и теперь <= ob_value  →  рынок выходит из перекупленности
```

Дополнительные фильтры:
- Прошёл `cooldown_seconds` с предыдущего входа (минимум 1 закрытая свеча × 5 мин)
- Прошёл `post_close_cooldown_seconds` после закрытия предыдущей позиции
- Время входа попадает в `trading_hours` (по умолчанию 10:05–18:30 МСК)

### Условие выхода по сигналу стратегии

```
LONG  → EXIT: arsi пересекает signal line сверху вниз (медвежий кросс)
SHORT → EXIT: arsi пересекает signal line снизу вверх (бычий кросс)
```

Выход блокируется если позиция удерживается меньше `min_hold_seconds`.

Как и в Combo, параллельно работают стоп-лосс, безубыток/трейлинг-стоп, тейк-профит и тайм-аут — они управляются `PositionManager` независимо от сигналов стратегии.

### Ключевые параметры

| Параметр | Назначение |
|---|---|
| `length` | Период RSI (по умолчанию 14) |
| `smooth` | Период сигнальной линии (по умолчанию 14) |
| `smo_type_rsi` | Метод сглаживания RSI: RMA / EMA / SMA / TMA (по умолчанию RMA) |
| `smo_type_signal` | Метод сглаживания сигнальной линии (по умолчанию EMA) |
| `ob_value` | Порог перекупленности (по умолчанию 80) |
| `os_value` | Порог перепроданности (по умолчанию 20) |
| `min_hold_seconds` | Минимальное время в позиции до выхода по кросс-сигналу (300 с = 1 свеча) |
| `cooldown_seconds` | Пауза между входами (рекомендуется кратно 300 с) |
| `stop_ticks` | Стоп-лосс в тиках (берётся `tick_size` из instruments.yaml) |
| `take_profit_ticks` | Тейк-профит в тиках (0 = отключён) |
| `trailing_stop_ticks` | Трейлинг-стоп в тиках (0 = отключён) |
| `max_hold_minutes` | Максимальное время удержания (тайм-аут) |

### Прогрев стратегии

RSI требует `length` закрытых свечей для корректного вычисления, плюс ещё `smooth` свечей для стабилизации сигнальной линии. При стандартных `length=14, smooth=14` стратегия начнёт генерировать сигналы примерно через **2 часа** после запуска (28 пятиминутных свечей). До прогрева входов не будет — это нормально.

## Управление в админке

### Раздел «Стратегии»

Страница `/strategies` позволяет включить или выключить каждую стратегию одной кнопкой.

**Важно:** выключение стратегии блокирует только открытие **новых** позиций. Уже открытые позиции продолжают работать — стоп-лосс, тейк-профит и тайм-аут срабатывают в штатном режиме. Это позволяет безопасно отключить стратегию в любой момент без принудительного закрытия позиций.

Состояние хранится в базе данных и переживает перезапуск бота.

### Раздел «Статистика»

На странице `/stats` доступен фильтр по стратегии. Карточки в верхней части показывают агрегированный P&L по каждой стратегии — удобно для сравнения их эффективности за любой период.

### Раздел «Сделки»

На странице `/trades` в фильтрах есть выпадающий список «Стратегия». В таблице добавлена колонка с тегом стратегии (COMBO / RSI), чтобы сразу видеть источник каждой сделки.

## Сравнение стратегий

| Характеристика | Combo (OFI+Print) | Augmented RSI 5m |
|---|---|---|
| Таймфрейм | тики (мс) | 5 минут |
| Источник сигнала | стакан + крупные сделки | цена закрытия свечей |
| Тип сигнала | продолжение импульса | разворот от экстремума |
| Прогрев | несколько минут | ~2 часа |
| Частота сделок | выше | ниже |
| Чувствительность к шуму | высокая (фильтруется подтверждениями) | низкая (5-мин бары) |
| Зависимость от ликвидности | высокая (нужен плотный стакан) | низкая |

Обе стратегии работают независимо на одних и тех же тикерах. Глобальный лимит позиций (`MAX_GLOBAL_POSITIONS`) применяется ко всем стратегиям вместе.
